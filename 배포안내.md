# 교회 홈페이지 템플릿 — 우리 교회용으로 만들기

이 폴더는 **여러 교회가 각자 자기 교회 홈페이지로 만들어 쓸 수 있는 템플릿**입니다.
아래 순서대로 따라 하면 됩니다. 대부분은 **파일 하나(`js/church.js`)만 고치면** 끝납니다.

> 원본 예시: 운평장로교회(k-logos.com). 교회 특정 정보는 모두 지워졌고,
> `○○교회` / `OOO CHURCH` 같은 **중립 표기**로 바뀌어 있습니다.

---

## ⚡ 빠른 시작 (핵심 3가지)

| 순서 | 무엇을 | 어디서 |
|---|---|---|
| 1 | 교회명·주소·전화·계좌·목사님 성함 | **`js/church.js`** ← 여기만 고치면 사이트 전체 반영 |
| 2 | 로고·아이콘 그림 교체 | **`images/`** 폴더 + `manifest.json` |
| 3 | 로그인·게시판 등 기능 켜기 | **`js/config.js`** (교회별 **새 계정** 발급) |

정관·소개글까지 바꾸려면 4~5단계도 진행하세요.

---

## 1단계 — 교회 기본 정보 (`js/church.js`) ★가장 중요★

`js/church.js` 파일을 메모장이나 편집기로 열어 값을 바꾸세요.
이 파일 하나가 **헤더·푸터·브라우저 제목·카카오톡 미리보기·헌금 계좌창**까지
사이트 전체에 자동 반영됩니다.

바꿀 항목:
- `name` : 한글 교회명 (예: `"믿음의교회"`)
- `nameEn` : 영문 교회명 (예: `"MIDEUM CHURCH"`)
- `since` : 설립 연도 (예: `"1988"`, 없으면 비워 둠)
- `tagline` : 한 줄 소개
- `address` / `phone` : 주소·대표 전화
- `pastors` : 섬기는 분들 (줄을 늘리거나 줄일 수 있음)
- `give` : 온라인 헌금 계좌 (은행·계좌번호·예금주). **안 쓰면 `account`를 비우면 버튼이 사라집니다.**
- `kakaoChannel` : 카카오톡 채널 홈 주소 (없으면 비움 → 버튼 사라짐)
- `domain` : 실제 홈페이지 주소 (검색·SNS 미리보기용)
- `themeColor` : 대표 색
- `seo` : (선택) 구글·네이버 검색 등록 인증코드

> 💡 값을 비워 두면 관련 버튼/기능이 **자동으로 숨겨집니다.** 잘못될 일이 없습니다.

## 2단계 — 로고·아이콘 교체

`images/` 폴더의 아래 그림들을 **우리 교회 로고로 교체**하세요.
(파일 이름·크기는 그대로 두고 내용만 바꿔야 합니다.)

| 파일 | 크기 | 쓰임 |
|---|---|---|
| `images/icon-192.png` | 192×192 | 헤더 로고·앱 아이콘 |
| `images/icon-512.png` | 512×512 | 카카오톡/검색 미리보기·앱 아이콘 |
| `images/icon-maskable-512.png` | 512×512 | 안드로이드 앱 아이콘(여백 포함) |
| `images/apple-touch-icon.png` | 180×180 | 아이폰 홈화면 아이콘 |
| `images/favicon-32.png` | 32×32 | 브라우저 탭 아이콘 |
| `로고.png` | 자유 | 예비 로고(선택) |

그리고 **`manifest.json`** 파일을 열어 아래 3줄의 `○○교회`를 실제 교회명으로 바꾸세요
(휴대폰에 앱으로 설치했을 때 뜨는 이름입니다):
```
"name": "○○교회",
"short_name": "○○교회",
"description": "○○교회 — 주보, 말씀, 예배 안내",
```

> 목사님 소개 사진은 `images/pastor-kim.jpg` (파일명 유지) 를 교체하세요.

## 3단계 — 기능 연결 (`js/config.js`) ★데이터 분리 필수★

로그인·게시판·푸시알림·파일업로드·재정관리 기능을 쓰려면
**우리 교회 전용 서비스 계정**을 새로 만들어 `js/config.js`에 넣어야 합니다.

> ⚠️ **절대 다른 교회의 값을 그대로 두지 마세요.**
> 예를 들어 남의 Supabase 주소를 그대로 두면 **교인·헌금·기도 자료가 서로 뒤섞이고 유출됩니다.**
> 반드시 아래처럼 우리 교회 것을 **새로** 발급하세요.

| 기능 | 만들 곳 | `config.js` 항목 |
|---|---|---|
| 로그인·게시판·나눔터 | [supabase.com](https://supabase.com) 새 프로젝트 → Settings ▸ API | `SUPABASE_URL`, `SUPABASE_ANON_KEY` |
| 푸시 알림 | [onesignal.com](https://onesignal.com) 새 앱 | `ONESIGNAL_APP_ID` |
| 사진·파일 업로드 | Cloudflare Workers + R2 (`cloudflare/` 참고) | `R2_UPLOAD_URL` |
| 재정·헌금조회 | Google Apps Script 웹앱 (`apps-script/` 참고) | `FINANCE_API_URL` |
| 도서관·연말정산 알림 | (선택) | `LIBRARY_*`, `FORMSUBMIT_EMAIL` |

- Supabase 데이터베이스 구조(테이블·보안정책)는 **`supabase/`** 폴더의 SQL을
  새 프로젝트의 SQL Editor에 붙여넣어 그대로 만들 수 있습니다.
- 비워 두면 그 기능만 "준비 중"으로 표시되고, **나머지 홈페이지는 정상 작동**합니다.
  (처음에는 비워 두고 사이트만 먼저 올려도 됩니다.)

## 4단계 — 정관 편집 (`bylaws.html`)

정관은 교회마다 내용이 다르므로 **`bylaws.html` 파일의 본문을 직접 편집**하세요.
- 교회명은 자동으로 채워집니다(`○○교회` → 우리 교회명).
- 하지만 **소재지 주소, 제정·개정 날짜, 조항 내용**은 직접 고쳐야 합니다.
  예) `제2조(소재지) 본 교회는 ○○로 00 …` → 실제 주소로 수정.
- 화면 상단의 "14장 90조" 같은 설명 문구도 우리 교회 정관에 맞게 수정하세요.

정관이 필요 없거나 형식이 완전히 다르면, 본문을 지우고 우리 정관을 붙여넣어도 됩니다.

## 5단계 — 소개·안내글 편집 (선택)

아래 페이지들은 교회마다 내용이 다른 **소개 글**입니다. 필요한 부분만 고치세요.
교회명·목사님 성함은 자동으로 채워지지만, **역사·연혁·선교 등 구체적 내용은 직접** 수정해야 합니다.

| 파일 | 내용 |
|---|---|
| `welcome.html` | 처음 오신 분 안내 · 예배 시간 · 오시는 길 · 교회 소개글 |
| `story.html` | 교회 연혁(설립 역사) · 섬김 부서 · 함께하는 공동체 |
| `word.html` | 이번 주 말씀 · 우리가 믿는 것(신앙고백) |
| `world.html` | 지역 연합사역 · 선교 |
| `privacy.html` | 개인정보처리방침 (교회명 자동, 연락처 확인) |

> 이 페이지들의 **연혁 날짜·인물·선교지** 등은 원본(운평교회) 예시가 일부 남아 있으니
> 우리 교회 내용으로 바꿔 주세요.

## 미리보기 (내 컴퓨터에서 확인)

이 폴더에서 아래 명령을 실행하면 브라우저로 미리 볼 수 있습니다.
```
python -m http.server 8000
```
→ 브라우저에서 `http://localhost:8000` 접속.

## 배포 (인터넷에 올리기)

- **GitHub Pages**(무료)를 권장합니다. 이 폴더를 새 저장소에 올리고 Pages를 켜면 됩니다.
- 자체 도메인을 쓰려면 `CNAME` 파일에 도메인을 적고(예: `mychurch.or.kr`), DNS를 연결하세요.
  (원본의 `CNAME`은 삭제되어 있습니다. 새로 만들어야 합니다.)
- `sitemap.xml`, `robots.txt`의 주소도 우리 도메인으로 맞추면 검색에 더 잘 노출됩니다.
